## 可选的方案

### 原有方案的优缺点、局限性及存在的问题

本项目的原有替代方案为自我判断化妆法与盲目跟风美妆博主方案，其核心问题如下：

#### 自我判断化妆法

优点：不需要额外工具，凭自我认知即可开始。缺点与局限性：普通人无法准确判断自己的脸型、眼距、三庭比例等面部特征；容易误判（如把圆脸判断为方脸）存在的问题：基于错误的自我认知化妆，效果往往适得其反

#### 盲目跟风美妆博主方案

优点：可免费获取大量化妆教程视频  
缺点与局限性：博主的面部特征与用户不同，同样的画法效果差异大；缺乏个性化指导；信息过载，不知道哪个教程适合自己存在的问题：无法解决 “适合别人的妆容不一定适合我” 的核心矛盾。

### 可重用的系统，与要求之间的差距

当前市场上可重用的同类系统为小红书 / 美妆社区类 APP，其与本项目要求的核心差距如下：

功能冗余度差距：小红书等平台以图文 / 视频分享为主，缺乏结构化的面部分析功能，用户需要自行搜索和判断，与本项目 “先分析再推荐” 的核心逻辑有本质区别。

技术架构差距：商业平台为复杂的前后端分离 + 云端服务架构，开发难度大、成本高，与本项目纯前端轻量化的开发要求不匹配。

使用门槛差距：商业平台需要用户注册登录、授权个人信息，本项目要求免登录、纯本地运行，保护用户隐私。

用途合规差距：商业平台代码为闭源商业版权，不可直接重用，且禁止非商业学习用途的修改与使用。

### 可选择的系统方案 1（最终建议方案）

方案名称：纯前端轻量化 Web 应用方案

方案核心内容：采用 HTML5+CSS3+JavaScript 纯前端技术栈开发，基于浏览器 Canvas 实现图片展示，LocalStorage 实现历史记录本地持久化，模拟 AI 面部分析流程，预设 12 项检测数据结构和妆容建议规则库。V2.0 规划接入腾讯云人脸分析 API 实现真实面部特征检测。

方案优势：开发难度低，适配开发人员技术能力；开发周期短，可在课程要求时间内完成；无任何开发与运营成本；免安装、免登录，用户使用门槛极低；数据纯本地存储，隐私安全性高；完全符合课程设计的需求与目标。

方案局限性：V1.0 面部分析为模拟数据，非真实 AI 检测；仅支持静态图片分析；历史记录仅能在当前浏览器本地存储，无法跨设备同步。

### 可选择的系统方案 2

方案名称：移动端微信小程序方案

方案核心内容：采用微信小程序框架开发，前端使用 WXML+WXSS+JS，调用微信原生 API 实现图片上传，接入微信云开发实现数据存储。

方案优势：移动端适配性好，可通过微信直接使用，用户触达便捷。

方案缺点：需要学习小程序开发框架，超出开发人员当前技术能力范围；需通过微信审核才能发布，周期不可控；与课程设计的 Web 应用开发要求不完全匹配。

### 选择最终方案的准则

本次最终方案的选择，严格遵循以下优先级从高到低的准则：

合规性准则：方案必须符合课程设计要求、软件工程国家标准、国家法律法规要求。

匹配性准则：方案必须与项目目标、功能需求、开发人员技术能力、开发周期完全匹配。可行性准则：方案必须具备可落地性，无不可突破的技术、成本、周期壁垒。成本效益准则：优先选择开发成本最低、开发效率最高的方案。用户体验准则：方案必须满足用户低使用门槛、高隐私安全、操作便捷的核心诉求。

基于以上准则，最终选定方案 1：纯前端轻量化 Web 应用方案作为本项目的落地实施方案。

## 所建议的系统

### 对所建议的系统的说明

本章节所建议的系统为 4.3 节选定的纯前端轻量化 Web 应用方案，完全匹配项目需求与可行性分析准则。

系统完全覆盖所有功能需求与非功能需求，实现「选择场景→上传照片→AI 面部分析→妆容建议→视频推荐→历史保存」的完整业务闭环。

系统采用三层分层架构设计：前端交互层负责页面渲染与用户操作接收；业务逻辑层负责场景筛选、面部分析模拟、妆容建议生成、历史记录管理、异常处理；数据存储层负责妆容规则数据、用户历史记录的本地读写与持久化。

系统所有核心能力均基于成熟前端技术实现，无自研复杂算法，开发难度与工作量完全适配开发人员的技术能力与项目周期要求。系统免后端服务、免注册登录、纯本地运行，无开发与运营成本，用户使用门槛极低，隐私安全性高。系统具备良好的可扩展性，后续可快速新增检测项、场景、妆容规则，无需重构核心业务代码。

### 数据流程和处理流程

5.2.1 核心数据流程

系统核心数据分为三类：

妆容规则数据：系统初始化时内置在 JavaScript 代码中，包含脸型 - 妆容映射规则、化妆重点 / 弱化点规则、视频关键词规则。

用户图片数据：用户上传图片后完成格式校验与大小校验，校验通过后通过 Canvas 渲染展示。

用户操作数据：用户的分析记录数据通过 LocalStorage 实现本地持久化存储，包含时间、场景、脸型、肤色、综合风格等信息。

5.2.2 核心处理流程

用户打开系统，完成页面初始化；用户选择目标场景；用户上传正面自拍照片，系统完成格式与大小校验；校验通过后，Canvas 显示原图；用户点击「开始面部分析」，系统执行模拟分析；分析完成后展示 12 项检测结果、妆容建议、B 站视频链接；自动保存分析记录至本地存储；用户可查看历史记录或点击重置按钮开启新一轮分析。
